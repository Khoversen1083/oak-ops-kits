# Resources

Source-of-truth files the coach reads while running the kit. Read-only,
not student-editable. If you need to change the template, fork the
repo.

| File | What it is |
|---|---|
| `monthly-audit-checklist.md` | The structured monthly audit per client. Uptime, automations run count, errors, time saved, value delivered, what broke, what improved. |
| `client-summary-email-template.md` | The monthly client-facing email. Three sections, plain language, written for the buyer not the tech person. |
| `internal-next-month-plan-template.md` | The internal plan per client. What to monitor, what to fix proactively, what to ship next, never sent to the client. |
| `upsell-trigger-checklist.md` | Signals that suggest an upsell is on the table. Triggers like "they asked for a second workflow", "their team grew", "they hit the value ceiling on the current build". |
| `voice-rules.md` | The hard and soft voice rules the coach enforces. Includes 5 worked bad-vs-good examples. |

The coach reads these in this order during a run:

1. `monthly-audit-checklist.md` plus `voice-rules.md` at Step 2
2. `client-summary-email-template.md` plus `voice-rules.md` at Step 3
3. `internal-next-month-plan-template.md` plus
   `upsell-trigger-checklist.md` at Step 4
