# Handoff doc template

A one-page, owner-facing document that ends 90 percent of the "is this
thing working?" texts. The most underrated deliverable in any
engagement. This is NOT the internal README. The handoff doc is for
the OWNER, plain English, no acronyms, no tool names.

## Rules

- Fill each section in the client's own words. Plain English. No tool
  names, no jargon.
- Make HOW TO RUN IT specific. A day, a time, a timezone. No "weekly".
- Make WHAT 'BROKEN' LOOKS LIKE concrete: a specific time the output
  should land, plus the promise that you get alerted before they do.
- Make WHEN TO ESCALATE a short bullet list, not a paragraph.
- Keep it to one page. If it spills over, you are over-explaining. Cut.
- If a line would confuse a 55-year-old founder, cut it.

## Template

```
[Client placeholder] AUTOMATION HANDOFF
Prepared by [Your business]
Effective Date: [date]

WHAT IT DOES

[1 to 2 sentences in their words. Example: "Every Monday morning, the
weekly status report is built automatically from your spreadsheet and
sent to your inbox. No manual lookup, no copy-paste, no rebuilding it
by hand."]

HOW TO RUN IT

[Most automations run themselves on a schedule. If yours does, write
the schedule here. Example: "It runs every Monday at 7am Eastern. You
do not need to do anything. If Monday is a holiday, it still runs."
If the client needs to trigger it, write the trigger step in one
sentence. Example: "When a new order shows up in your orders sheet,
the summary fires within 10 minutes."]

WHAT YOU SHOULD SEE

[Concrete description of the output. Example: "An email with subject
line 'Weekly Status Report, [date]'. The body has 3 sections: new
items this week, items awaiting review, items closed. Numbers are
current as of Sunday at midnight."]

WHAT 'BROKEN' LOOKS LIKE

[Concrete signals of failure. Example: "If the report does not arrive
by 8am Monday, something has gone wrong. You do not need to do
anything. I get alerted before you do, will have it fixed and
explained to you by 10am. If you are seeing the report but the
numbers look wrong (last week's data instead of this week's), email
me directly."]

WHO TO CALL

[Your name]
[Your business]
[email]
[Response time, example: "I check this address daily. For anything
urgent, email is fastest, I'll have a reply inside 4 hours during
business days."]

WHEN TO ESCALATE

[Plain instructions, 3 to 5 lines max. Example:
- To pause the automation: email me, paused within 4 hours, no
  technical work on your end.
- To change what's in the report (add a column, remove a section,
  change the time): email me, most adjustments are 30 minutes of
  work and are included in the monthly.
- To change the recipient list: email me with the new addresses.
- If you do not hear back within 24 hours, text me at [number].]

WHAT THE MONTHLY COVERS

[Plain English. Example:
- Monitoring (I get the alerts, not you)
- Fixes when something breaks
- Adjustments when your process changes
- A 15 to 30 minute monthly check-in
- Priority response within 24 hours]

WHAT IT DOES NOT COVER (without separate scope)

[Plain English. Example:
- Building a new, separate automation
- Adding integrations to systems we did not already discuss
- Major redesigns of the existing report]
```

## Send-day email (same day as the walkthrough)

```
Subject: [Project name] handoff document

Hey [first name],

Attached: a one-page handoff doc for the automation we just walked
through. Save this somewhere you can find it. It explains how to use
it, what to do if something looks wrong, and what's covered in the
monthly.

First scheduled run is [day, date, time].

Talk [next contact day].

[your first name]
```

## Common mistakes

- Wrote it for yourself, not the owner. If a line has a tool name or
  an acronym, cut it.
- Vague schedule. "Weekly" is not a schedule. "Monday 7am Eastern" is.
- Skipped WHAT 'BROKEN' LOOKS LIKE. That is the section that kills
  the panic texts. Never skip it.
- Two pages. The owner loses it. One page, or cut.
- No retainer scope. If they do not know what is covered, every
  request lands as urgent and unpaid.
